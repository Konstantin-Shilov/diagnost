Рефактори код следуя принципам проекта:

1. **DRY (Don't Repeat Yourself)**
   - Вынеси дублирующийся код в утилиты/компоненты
   - Переиспользуй существующие компоненты

2. **Разделение ответственности**
   - Бизнес-логика → services/
   - UI компоненты → components/
   - Общее → common/
   - Состояние → store/

3. **Type Safety**
   - Убери any
   - Добавь типы для всех Props
   - Используй Zod для валидации где нужно

4. **Производительность**
   - Мемоизация где необходимо (useMemo, useCallback)
   - Оптимизация ререндеров
   - Code splitting если компонент большой

5. **Читаемость**
   - Понятные имена переменных/функций
   - Декомпозиция сложных функций
   - Комментарии для неочевидной логики
